## 二、产品概述
### 2.1 业务背景与核心痛点
当前数字化中心大数据业务面临4大核心痛点，急需通过自研一体化平台解决：
1.  **数据入湖效率极低**：现有数据入湖依赖人工开发SQL、配置任务，当下有200张表紧急入湖需求，人力严重不足，无自动化配置能力；
2.  **运维能力分散被动**：大数据组件（DolphinScheduler、StreamPark、Flink CDC等）、物理机/虚拟机监控分散在多个平台，无统一看板、告警机制，故障需等待业务方反馈，无主动预警能力；依赖第三方运维平台，需求迭代受限于其他团队，边界耦合严重；
3.  **平台价值无法量化**：无统一的报表访问、接口调用、视图查询统计能力，无法向管理层量化数据平台的业务价值与使用情况；
4.  **能力拓展性不足**：现有平台无原生AI能力集成空间，数据治理、指标管理等进阶需求无统一承载载体。

### 2.2 产品目标
#### 短期目标（Q1，MVP核心交付）
1.  完成平台基础框架与统一权限体系搭建；
2.  落地数据入湖自动化能力，解决200张表批量入湖的紧急需求，提升数据开发人效80%以上；
3.  实现核心大数据组件与资源的统一运维监控，完成被动运维向主动运维转型。

#### 长期目标（Q2及以后）
1.  完成全模块能力落地，打造自研的大数据全生命周期一体化管理平台；
2.  原生集成AI问数、数据治理、指标库等进阶能力，实现平台能力的全面升级；
3.  实现平台降本增效，通过资源监控优化冗余服务器资源，通过自动化能力降低人力成本。

### 2.3 目标用户与角色
| 用户角色 | 核心诉求 |
|----------|----------|
| 大数据开发工程师 | 自动化完成数据入湖、数仓开发任务，减少重复人工操作，统一管理开发任务 |
| 大数据运维工程师 | 统一监控服务器、大数据组件、任务运行状态，快速接收告警、定位故障，统一权限管控 |
| 数据分析师/业务人员 | 快速获取标准化的宽表数据，后续可通过AI问数快速完成数据查询 |
| 部门管理者 | 查看平台运行情况、业务访问数据，量化平台价值，完成工作汇报 |

### 2.4 需求范围与边界
#### 纳入范围
1.  平台基础：统一用户权限体系、微服务脚手架框架；
2.  核心模块：数据入湖自动化、一体化运维监控、平台价值统计分析；
3.  进阶模块：数据治理、AI问数、指标库管理（后续迭代落地）。

#### 不纳入范围
1.  替代底层大数据组件（HDFS、Doris、Flink等），仅做统一管理与集成；
2.  业务系统的底层数据改造，仅做数据同步与接入；
3.  第三方运维平台的深度定制开发，仅做能力参考与自研适配。

## 三、核心业务流程
### 3.1 离线数据入湖核心流程
数据源配置 → 表勾选与同步规则配置（全量/增量）→ 系统自动生成同步SQL → 定时任务与依赖配置 → 任务并行/串行执行 → 任务运行监控 → 数据入湖完成

### 3.2 运维监控核心流程
监控对象接入（服务器/组件/任务）→ 指标数据采集 → 统一看板可视化 → 告警规则配置 → 异常触发告警 → 故障处理与记录

### 3.3 平台整体业务架构
```
底层数据源 → 数据入湖自动化模块 → 数据湖/数仓存储 → 数据开发与治理 → Doris数据服务 → 上层应用（BI报表、API服务、AI问数）
↓
统一权限体系 + 一体化运维监控 + 统计分析
```

## 五、功能需求详情
### 5.1 统一权限管理模块（基础底座）
#### 功能概述
平台唯一的用户与权限管控中心，实现一套账号体系管控全平台所有模块与功能，是所有模块的前置依赖。

#### 核心功能
1.  **用户管理**：支持账号新增、编辑、禁用、删除，支持绑定用户角色与所属项目；
2.  **角色管理**：预置管理员、开发、运维、分析师、访客5种默认角色，支持自定义角色与权限点配置；
3.  **权限管控**：
    - 功能权限：管控菜单、按钮的可见与操作权限；
    - 数据权限：管控项目、数据源、数据表的访问与操作权限；
    - 运维权限：管控服务器、组件的监控与操作权限；
4.  **单点登录**：支持一套账号登录平台所有模块，支持嵌入的第三方组件页面免登适配。

#### 业务规则
- 平台超级管理员拥有所有权限，可分配其他账号权限；
- 权限配置遵循最小权限原则，默认新账号无任何权限，需手动分配；
- 所有操作需留痕，支持操作日志查询与追溯。

---

### 5.2 数据入湖自动化模块（P0核心）
#### 功能概述
可视化、自动化完成业务数据到数据湖的同步任务开发与执行，解决批量表入湖的人力瓶颈问题，是本次迭代的核心交付内容。

#### 核心功能
1.  **数据源管理**
    - 支持可视化配置关系型数据源（PG、TiDB、MySQL）的连接信息，支持连接测试；
    - 支持自动读取数据源中的表清单、表结构、字段信息；
    - 支持数据源的新增、编辑、删除、权限分配。
2.  **入湖任务配置**
    - 支持批量勾选需要入湖的数据表，支持按库批量选择；
    - 支持同步规则配置：全量同步（每日清空重写）、增量同步（仅同步增删改变化数据）；
    - 支持系统自动生成标准的入湖同步SQL，支持人工编辑调整；
    - 支持定时任务配置：设置执行周期（小时/天/周）、执行时间；
    - 支持任务依赖配置：可视化设置任务的上下游依赖关系，支持串行/并行执行配置，支持并行度调整；
    - 支持批量生成任务，按规则自动分组创建任务。
3.  **任务运维中心**
    - 支持查看所有任务的执行状态、执行历史、运行日志；
    - 支持任务的手动触发、暂停、恢复、终止操作；
    - 支持任务执行失败重试配置，失败自动触发告警通知。

#### 业务规则
- 全量同步默认每日凌晨执行，增量同步最小周期支持1小时；
- 下游任务必须等待所有上游依赖任务执行成功后，方可触发执行；
- 入湖任务默认写入数据湖HDFS，支持映射到Doris表。

### 5.3 一体化运维监控模块（P1高优）
#### 功能概述
整合服务器、大数据组件、任务的全链路监控能力，实现统一看板、统一告警、统一运维操作，解决现有运维分散、被动的问题。

#### 核心功能
1.  **基础设施监控**
    - 接入物理机/虚拟机的核心监控指标：CPU使用率、内存使用率、磁盘占用率、网络带宽与丢包率；
    - 支持服务器资源的可视化看板，支持峰值、均值数据查看，支持扩容预警；
    - 基于普罗米修斯做指标采集，Grafana做看板可视化，支持嵌入平台统一展示。
2.  **组件服务监控**
    - 支持大数据组件（DolphinScheduler、StreamPark、Flink、报表服务等）的运行状态监控；
    - 支持查看组件的资源占用、节点健康度、服务运行时长；
    - 支持嵌入组件原生运维web页面，实现平台内一站式运维操作；
    - 支持组件服务异常状态监控与告警。
3.  **任务运行监控**
    - 统一查看全平台同步任务、调度任务的运行状态、成功率、耗时排行；
    - 支持失败任务的快速筛选与定位，支持查看失败原因与日志。
4.  **告警中心**
    - 支持自定义告警规则：监控指标阈值、告警级别、触发条件；
    - 支持多渠道告警通知：短信、企业微信/钉钉、邮件；
    - 支持告警事件的记录、处理、闭环全流程追溯。

#### 业务规则
- 监控数据采集延迟不超过1分钟；
- 核心指标告警触发后，必须在1分钟内完成通知推送；
- 所有运维操作需留痕，支持操作日志查询。

---

### 5.4 平台价值统计分析模块（P1高优）
#### 功能概述
统计平台核心资产与业务访问数据，形成可视化看板，量化平台业务价值，支撑管理层工作汇报。

#### 核心功能
1.  **资产统计**
    - 平台核心资产统计：报表总数、PC端/移动端报表分布、数据湖表总数、接入数据源总数；
    - 支持按时间维度查看资产增长趋势。
2.  **访问统计**
    - 报表/看板的每日访问量、访问人数、高频访问排行；
    - Doris视图/接口的查询次数、访问IP、调用频次统计；
    - 支持按时间、业务板块筛选查看访问数据。
3.  **可视化看板**
    - 预置管理层汇报看板，核心指标可视化展示；
    - 支持看板的导出、分享功能。

#### 业务规则
- 访问数据支持T+1更新，核心指标支持准实时更新；
- 支持按业务板块、权限范围做数据隔离，不同角色仅能查看权限内的统计数据。

### 5.5 进阶功能模块（P2后续迭代）
1.  **数据治理模块**：数据表与字段的分类分级、安全打标、敏感字段自动加密、数据质量校验；
2.  **AI问数模块**：场景化知识库搭建、自然语言转SQL查询、可视化图表自动生成、API能力对外暴露；
3.  **指标库模块**：业务指标统一管理、指标定义标准化、指标改动记录全追溯、指标数据一键查询；
4.  **实时入湖模块**：适配Flink CDC、Citino等实时日志捕获工具，可视化配置实时数据同步任务。

## 六、非功能需求
1.  **性能需求**
    - 数据入湖任务配置页面响应时间≤2秒，列表查询响应时间≤1秒；
    - 单批次支持至少500张表的入湖任务配置与并行执行；
    - 监控看板数据刷新延迟≤1分钟，告警触发延迟≤1分钟；
    - Doris查询统计数据的响应时间≤3秒。
2.  **安全性需求**
    - 所有用户密码需加密存储，禁止明文存储；
    - 所有操作需做权限校验，禁止越权访问与操作；
    - 数据源连接信息中的账号密码需加密存储，支持脱敏展示；
    - 支持操作日志全留存，留存时间不少于6个月。
3.  **兼容性需求**
    - 支持Chrome、Edge等主流浏览器最新2个版本；
    - 适配PC端主流分辨率，支持响应式布局；
    - 支持与现有大数据组件（HDFS、Doris、DolphinScheduler等）的稳定兼容。
4.  **可扩展性需求**
    - 采用微服务架构，支持各模块独立部署、独立迭代；
    - 预留扩展接口，支持后续AI能力、数据治理等模块的快速接入；
    - 支持数据源类型的扩展，可快速适配新的数据源接入。
5.  **可用性需求**
    - 平台核心服务可用性≥99.9%；
    - 支持服务降级与容错，单模块故障不影响其他模块正常运行；
    - 支持任务失败重试、断点续传，保障数据同步的稳定性。

4.  批量入湖：可一次性完成200张表的入湖任务配置，任务可按规则并行/串行执行，成功率≥99%。

### 7.2 其他功能验收标准
1.  运维监控：可成功接入服务器监控数据，可视化看板正常展示，可配置告警规则，异常触发后可正常推送告警通知；可嵌入组件运维页面，实现一站式操作；

1.  页面响应时间符合需求，核心操作无明显卡顿；
2.  平台连续72小时运行无核心服务故障，可用性达标；

| 技术风险 | 现有大数据组件版本多样，集成适配难度大 | 中 | 1. 迭代0完成组件版本调研与适配测试；2. 先采用web页面嵌入的方式实现快速集成，后续再做深度接口适配 |
| 资源风险 | 开发人力不足，无法按迭代计划完成交付 | 中 | 1. 严格控制迭代范围，优先保障P0/P1需求；2. 非核心需求延后至后续迭代；3. 协调团队资源，优先保障核心模块开发 |
| 需求变更风险 | 迭代过程中新增需求，影响原有交付计划 | 中 | 1. 严格遵循敏捷变更流程，新增需求纳入需求池，下个迭代评估排期；2. 紧急需求需评估对当前迭代的影响，经负责人审批后调整 |
